Getting Started with rfsrs

Introduction

rfsrs provides R bindings for the FSRS (Free Spaced Repetition Scheduler) algorithm. FSRS is a modern spaced repetition algorithm that models human memory more accurately than traditional algorithms like SM-2.

library(rfsrs)

Understanding FSRS

The Memory Model

FSRS models memory using two key variables:

  1. Stability (S): Represents how long a memory will last. Higher stability means slower forgetting. Specifically, stability is the number of days until retrievability drops to 90%.

  2. Difficulty (D): Represents how hard the material is to learn. Ranges from 1-10. Higher difficulty means stability grows more slowly with each review.

Retrievability

Retrievability is the probability that you can recall a memory at a given time.

# Visualize the forgetting curve for different stability values
stability_values <- c(1, 5, 10, 30)
days <- 0:60

# Simple base R plot
plot(NULL, xlim = c(0, 60), ylim = c(0, 1),
     xlab = "Days since last review", ylab = "Retrievability",
     main = "Forgetting Curves for Different Stability Values")
colors <- c("red", "orange", "blue", "green")
for (i in seq_along(stability_values)) {
  s <- stability_values[i]
  r <- sapply(days, function(d) fsrs_retrievability(s, d))
  lines(days, r, col = colors[i], lwd = 2)
}
legend("topright", legend = paste("S =", stability_values, "days"), 
       col = colors, lwd = 2)
abline(h = 0.9, lty = 2, col = "gray")

Basic Usage

Creating a New Card

When you first learn a card, you create an initial memory state based on your first rating:

# Rating scale: 1=Again, 2=Hard, 3=Good, 4=Easy
state <- fsrs_initial_state(rating = 3)  # First review, rated "Good"
print(state)
#> $stability
#> [1] 2.3065
#> 
#> $difficulty
#> [1] 2.118104

The initial stability depends on your rating:

ratings <- 1:4
rating_names <- c("Again", "Hard", "Good", "Easy")

for (i in ratings) {
  s <- fsrs_initial_state(rating = i)
  cat(sprintf("%s (rating=%d): stability=%.2f, difficulty=%.2f\n",
              rating_names[i], i, s$stability, s$difficulty))
}
#> Again (rating=1): stability=0.21, difficulty=6.41
#> Hard (rating=2): stability=1.29, difficulty=5.11
#> Good (rating=3): stability=2.31, difficulty=2.12
#> Easy (rating=4): stability=8.30, difficulty=1.00

Checking Retrievability

After a review, you can check how well youâ€™ll remember the card over time:

state <- fsrs_initial_state(rating = 3)

cat("Retrievability after:\n")
#> Retrievability after:
for (days in c(1, 3, 7, 14, 30)) {
  r <- fsrs_retrievability(state$stability, elapsed_days = days)
  cat(sprintf("  %2d days: %.1f%%\n", days, r * 100))
}
#>    1 days: 95.3%
#>    3 days: 87.5%
#>    7 days: 76.4%
#>   14 days: 64.2%
#>   30 days: 49.7%

Getting the Next Interval

FSRS calculates the optimal interval for your next review based on desired retention:

state <- fsrs_initial_state(rating = 3)

# Get interval for 90% desired retention (default)
interval_90 <- fsrs_next_interval(state$stability, desired_retention = 0.9)
cat(sprintf("Interval for 90%% retention: %.1f days\n", interval_90))
#> Interval for 90% retention: 2.3 days

# Get interval for 85% desired retention (fewer reviews)
interval_85 <- fsrs_next_interval(state$stability, desired_retention = 0.85)
cat(sprintf("Interval for 85%% retention: %.1f days\n", interval_85))
#> Interval for 85% retention: 4.4 days

# Get interval for 95% desired retention (more reviews)
interval_95 <- fsrs_next_interval(state$stability, desired_retention = 0.95)
cat(sprintf("Interval for 95%% retention: %.1f days\n", interval_95))
#> Interval for 95% retention: 0.9 days

Simulating a Review

After reviewing a card, update the memory state:

# Initial state
state <- fsrs_initial_state(rating = 3)
cat(sprintf("Initial: stability=%.2f\n", state$stability))
#> Initial: stability=2.31

# Review after 2 days, rate "Good"
state <- fsrs_next_state(
  stability = state$stability,
  difficulty = state$difficulty,
  rating = 3,
  elapsed_days = 2
)
cat(sprintf("After review 1: stability=%.2f\n", state$stability))
#> After review 1: stability=2.31

# Review after 5 days, rate "Good"
state <- fsrs_next_state(
  stability = state$stability,
  difficulty = state$difficulty,
  rating = 3,
  elapsed_days = 5
)
cat(sprintf("After review 2: stability=%.2f\n", state$stability))
#> After review 2: stability=2.31

Simulating Long-term Learning

Letâ€™s simulate learning a card over many reviews:

simulate_learning <- function(n_reviews = 10, rating = 3, desired_retention = 0.9) {
  state <- fsrs_initial_state(rating = rating)
  
  results <- data.frame(
    review = 0,
    day = 0,
    stability = state$stability,
    difficulty = state$difficulty,
    interval = NA
  )
  
  current_day <- 0
  
  for (i in 1:n_reviews) {
    interval <- fsrs_next_interval(state$stability, desired_retention)
    current_day <- current_day + interval
    state <- fsrs_next_state(state$stability, state$difficulty, interval, rating)
    
    results <- rbind(results, data.frame(
      review = i,
      day = current_day,
      stability = state$stability,
      difficulty = state$difficulty,
      interval = interval
    ))
  }
  
  return(results)
}

# Simulate 10 reviews, always rating "Good"
learning <- simulate_learning(n_reviews = 10, rating = 3)
print(learning)

Summary

Key functions:

Function Description
fsrs_default_parameters() Get the 21 default FSRS-6 parameters
fsrs_initial_state(rating) Create initial memory state for a new card
fsrs_next_state(stability, difficulty, rating, elapsed_days) Calculate next state after review
fsrs_next_interval(stability, desired_retention) Get optimal interval for desired retention
fsrs_retrievability(stability, elapsed_days) Calculate probability of recall
